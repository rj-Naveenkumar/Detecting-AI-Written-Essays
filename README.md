# AI-Generated Essay Detection

## Overview
This project aims to identify whether a given essay is authored by a human or generated by an AI language model like GPT-2. It leverages machine learning techniques to analyze textual features and classify essays as either human-written or AI-generated.

---

## Features
- Data Collection: Uses a mix of GPT-2 generated essays and human-written essays from reliable sources like Wikipedia.
- Machine Learning Model: Implements a text classification model for detecting authorship.
- Preprocessing: Includes cleaning, tokenization, and feature extraction from essays.
- Evaluation: Provides detailed metrics to assess the performance of the model.

---

## Tech Stack
- Programming Language: Python
- Machine Learning Libraries: TensorFlow, Scikit-learn
- Text Processing Tools: NLTK, SpaCy
- Data Handling: Pandas, NumPy

---

###  Observations
- The accuracy suggests that the model might be guessing randomly in this binary classification problem.
- Possible reasons for this result include:
  1. Limited and example dataset used for training.
  2. Insufficient preprocessing or feature extraction from the text.
  3. Imbalanced dataset leading to suboptimal model performance.

---

##  Dataset Overview

The dataset used for this project is currently a small example dataset designed for initial testing and proof-of-concept purposes. It includes essays labeled as either AI-generated or human-written.

###  Dataset Details
- Categories:
  - `Human-Written`: Essays written by humans.
  - `AI-Generated`: Essays generated by AI tools.
- Sample Size: Limited to a small number of essays for initial testing.
- Features:
  - Raw text of the essay.
  - Corresponding label (`AI-generated` or `Human-written`).

###  Limitations
1. Small Dataset: Not large enough for robust training and evaluation.
2. Imbalanced Classes: Potential skew in the distribution of human-written and AI-generated essays.
3. Simplistic Features: Text representation relies on basic techniques, limiting the model's understanding.

###  Future Dataset Expansion
To enhance the model's accuracy and generalization:
1. Human-Written Essays: Collect data from reliable sources like Wikipedia or academic essays.
2. AI-Generated Essays: Generate essays using GPT-2 or similar language models.
4. Preprocessed Features:
   - Word embeddings (e.g., Word2Vec, GloVe, BERT).

---

##  Methods Used

### 1. Data Preprocessing
   - Text Cleaning: Removal of punctuation, special characters, and numerical values to standardize the text.
   - Normalization: Conversion of all text to lowercase for uniformity.
   - Tokenization: Splitting essays into individual words or meaningful units.
   - Feature Extraction: Converting text into numerical form using basic methods like:
     - Bag-of-Words (BoW)

### 2. Machine Learning Model
   - Binary classification to label essays as either AI-generated or human-written.

### 3. Loss Function and Optimization
   - Binary Cross-Entropy Loss: Used for evaluating the performance of binary classification.
   - Optimizer: Adam optimizer to update weights during training.

### 4. Evaluation Metrics
   - Accuracy: Used to assess the model's ability to correctly classify essays.
   - Loss Monitoring: Observing the training and validation loss to detect overfitting or underfitting.

---

##  Results (Current Status)

At this initial stage, the model achieved a validation accuracy of 50% across all epochs.  
This indicates that the model is not yet effectively learning patterns to distinguish between AI-generated and human-written essays.
